# Phase 5: Guided Project Track - Context

**Gathered:** 2026-01-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Students learn by building THEIR app instead of following generic curriculum. The wizard guides from vague idea to scoped MVP, curriculum adapts to their project type, and they ship something real they care about while learning Claude Code fundamentals.

This phase does NOT build a separate feature - it's an alternative learning path where project work IS the teaching mechanism.

</domain>

<decisions>
## Implementation Decisions

### Discovery Wizard Entry Point
- Start open-ended: "Tell me about your idea"
- Conversational and welcoming tone (not categorical/directive)
- Let students explain in their own words before guiding toward structure
- No example projects shown first - let their authentic idea emerge

### Scope Reduction Flow
- Dream first, THEN narrow
- Ask: "What features would the full version have?"
- Acknowledge and validate the complete vision
- THEN guide to V1 scope using value-ranking questions
- Pattern: Validate → Acknowledge → Scope (not deflate immediately)

### V1 Feature Constraint
- Hard limit: 3 features maximum, no exceptions
- GSD-style discipline applied to beginner projects
- "Features" definition: Distinct user-facing capabilities
- If student proposes 5-7 ideas, wizard MUST help reduce to 3
- No flexibility based on complexity or time estimates

### Prioritization Method
- Value ranking approach: "Which 3 make it useful even without the others?"
- Focus on core value proposition
- Test: "Would this be useful with ONLY these 3 features?"
- GSD-style ruthless prioritization, adapted for encouraging tone
- The MVP should solve the problem even in minimal form

### Claude's Discretion
- Specific wizard questions/phrasing (as long as principles above are followed)
- How to handle edge cases (e.g., student insists on 5 features - gentle but firm)
- Wizard UI/UX presentation in teaching flow
- Error messages and validation feedback

</decisions>

<specifics>
## Specific Ideas

- GSD principles inspire the wizard design (scoping, incremental delivery, MVP focus)
- Wizard should feel like a helpful coach, not a gatekeeper
- Balance: Strict on scope (3 features), encouraging in tone
- The constraint exists to ensure students finish, not to limit creativity

</specifics>
