  **Add/Update "Updating the Living Cheat Sheet" subsection:**
  This subsection defines what happens after each lesson completion:

  When to Update:
  - After lesson complete (not after every task)
  - Only if lesson has valuable reference content (commands, shortcuts, insights)

  What to Append:
  1. Commands learned - list with brief descriptions
  2. Key insights - 1-3 takeaways with light bulb emoji
  3. Copy-paste examples - ready-to-use code blocks
  4. Common mistakes - pitfalls to avoid
  5. Pro tips - advanced usage patterns

  How to Update:
  1. Read current MY_CHEAT_SHEET.md
  2. Update header stats (level, XP, current module)
  3. Append new section under the appropriate module heading:
     ```markdown
     #### Lesson X.Y: [Lesson Name]
     <new content organized by category>
     ```
  4. Write updated MY_CHEAT_SHEET.md
  5. Regenerate MY_CHEAT_SHEET.html by reading the current HTML file, inserting the new markdown content converted to HTML within the existing styled template, and writing the updated HTML
  6. Show student: "Cheat sheet updated with [Lesson Name]! View: open MY_CHEAT_SHEET.html"

  Example Before/After (showing a real update):
  ```
  BEFORE (MY_CHEAT_SHEET.md after Module 1, Lesson 1.3):
  ### Module 1: Terminal Basics
  #### Lesson 1.1: Your First Command
  - `pwd` - print working directory (where am I?)
  #### Lesson 1.2: Looking Around
  - `ls` - list files and folders
  - `ls -la` - list with details and hidden files

  AFTER (appended for Lesson 1.3):
  #### Lesson 1.3: Moving Around
  - `cd foldername` - change into a folder
  - `cd ..` - go up one level
  - `cd ~` - go to home directory
  - Pro tip: Use Tab to autocomplete folder names
  ```

  Content Guidelines:
  - Keep entries concise (1-2 lines max)
  - Use code formatting for commands
  - Group related commands together
  - Prioritize practical reference over theory
  - Include real examples they can copy-paste
  - No filler - only useful reference material
