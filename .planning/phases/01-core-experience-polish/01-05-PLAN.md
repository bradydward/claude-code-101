  <action>
  Read CLAUDE.md Sections 3, 4, 6, and 11. Add a new Section (or subsection to Section 21) called "### Game Mechanics Calculation Rules" with explicit formulas and worked examples:

  **1. Task Completion XP Calculation:**
  ```
  base_xp = 10
  class_bonus_xp = base_xp * class_xp_modifier
    - Gigachad: +20% when task involves creating files/features -> 12 XP
    - Sigma: no XP bonus (bonus is on streaks)
    - Others: no XP bonus
  total_task_xp = base_xp + class_bonus_xp (rounded down)
  ```

  **2. Stat Growth Per Task:**
  ```
  stat_tag = current lesson's stat_tag (Speed, Accuracy, Creativity, Efficiency)
  base_stat_gain = +1 to stat_tag stat
  class_primary_bonus = +1 to class primary stat (ALWAYS, every lesson)
    - Gigachad: +1 Creativity (always)
    - Sigma: +1 Speed (always)
    - Aura Farmer: +1 Aura (extra, on top of base +1)
    - Chaos Agent: +1 Creativity AND +1 Speed (both)
    - Meme Lord: +1 Creativity (always)
    - Hackerman: +1 Efficiency (always)
  aura_gain = +1 base (all classes) + class_aura_bonus
    - Aura Farmer: +2 Aura per lesson (not +1)
    - Others: +1 Aura per lesson
  ```

  **3. Streak Freeze Logic:**
  ```
  On session start:
  1. Read last_session date
  2. Calculate days_gap = today - last_session
  3. If days_gap == 0: same day, no streak change
  4. If days_gap == 1: consecutive day, streak_days += 1
  5. If days_gap == 2 AND streak_freeze_available == true:
     - Auto-use freeze: streak_days += 1 (preserved!)
     - Set streak_freeze_available = false
     - Display: "Streak Freeze used! Your streak is safe."
  6. If days_gap == 2 AND streak_freeze_available == false:
     - Streak broken: set streak_days = 1
     - Display encouragement: "Your X-day streak ended. That's still impressive!"
  7. If days_gap > 2: streak broken regardless
     - Set streak_days = 1
     - Track longest_streak if current > longest

  Freeze reset:
  - Track last_freeze_reset date
  - If new week (Monday): streak_freeze_available = true
  - Sigma Grinder class: Gets extra freezes via skill tree (check skills_unlocked)
  ```

  **4. Skill Unlock Stat Bonuses:**
  ```
  When student unlocks a skill:
  1. Read skill from skill_trees.json by skill_id
  2. Apply stat_bonus: stats[skill.stat] += skill.bonus_value
  3. Add skill_id to skill_tree.skills_unlocked array
  4. Decrement skill_tree.points_available by 1
  5. Display skill unlock celebration (VIS-06)
  ```

  **5. Aura Economy:**
  ```
  Earning:
  - Task complete: +1 Aura (base)
  - Aura Farmer class: +2 Aura per lesson (replaces base +1 on lesson-level)
  - Module complete: +10 Aura
  - Streak milestones: +5 (3 days), +10 (7 days), +20 (14 days), +50 (30 days)
  - Easter eggs: +5 to +20 Aura

  Spending:
  - Deducts from current_balance only
  - total_earned NEVER decreases (used for glow/reputation)
  - Glow calculated from total_earned, not balance

  Glow thresholds: 0=none, 25=faint, 75=soft, 150=bright, 300=radiant, 500=blinding, 1000=transcendent
  Reputation: 0=Newcomer, 50=Known, 150=Respected, 300=Famous, 600=Legendary, 1000=Mythical, 2000=Transcendent
  ```

  **6. Easter Egg Trigger:**
  ```
  After 4th lesson in a single day (daily_lessons.completed_today >= 4):
    - Trigger easter egg (max 1 per week, check easter_eggs.last_triggered)
    - Award +25-50 XP and +5-20 Aura
    - Chaos Agent class: triggers at 3 lessons instead of 4
    - Chaos Agent class: rewards doubled
  ```

  Document these as the AUTHORITATIVE calculation rules. When in doubt, these formulas are correct.
  </action>

  - Skill unlock process is step-by-step
  - Aura economy distinguishes earning vs spending vs glow
  - Easter egg trigger includes Chaos Agent exception
  - All formulas use specific numbers (not vague "bonus")
  </verify>
