**SCOPE CLARIFICATION:** This phase adds background music to the **web portal onboarding experience** (web/terminal.html) only. The main CLI teaching sessions (where students run `claude` command and complete the 15-module curriculum) will receive background music in a future phase. This separation is intentional - web portal uses browser-based Howler.js, CLI uses system-based afplay for event sounds. See Phase 5+ for CLI background music planning.

<task type="auto">
  <name>Task 1: Create BackgroundMusicManager class</name>
  <files>web/js/music-system.js</files>
  <action>
Create `web/js/music-system.js` with a `BackgroundMusicManager` class implementing:

**Constructor:**
- `this.currentTrack = null` (Howl instance)
- `this.isUnlocked = false` (autoplay gate)
- `this.preferences = this.loadPreferences()` (from localStorage)
- `this.trackLibrary = null` (loaded from metadata.json)

**Methods:**

1. `async init()` - Load metadata.json via fetch, check autoplay policy via `navigator.getAutoplayPolicy` (with fallback for older browsers), prepare default track muted.

2. `loadTrack(trackId, startMuted = false)` - Call `this.currentTrack.unload()` if exists (CRITICAL for memory). Create new Howl with `html5: true`, `loop: true`, volume from preferences (or 0 if muted). Handle `onload` and `onloaderror` callbacks.

3. `unlockAudio()` - On first user interaction, play a silent base64 WAV to unlock Web Audio API context. Set `isUnlocked = true`. If preferences.enabled, fade in current track to preferences.volume over 2s.

4. `play()` / `pause()` - Start/stop current track. Only play if `isUnlocked` and track loaded.

5. `switchTrack(trackId)` - Fade out current (2s), then load and fade in new track (2s). Use setTimeout for the crossfade timing.

6. `setVolume(volume)` - Update Howl volume AND save to preferences.

7. `toggle()` - Flip `preferences.enabled`, play or pause accordingly, save preferences.

8. `loadPreferences()` - Read from `localStorage.getItem('music_preferences')`. Defaults: `{ track: 'chill-lofi', volume: 0.3, enabled: true }`.

9. `savePreferences()` - Write to localStorage as JSON.

10. `destroy()` - Call `currentTrack.unload()`. Called on page beforeunload.

**Module-level initialization at bottom of file:**
```javascript
const musicManager = new BackgroundMusicManager();

document.addEventListener('DOMContentLoaded', () => {
  musicManager.init();
});

// Unlock on first user interaction (click, keypress, or touch)
['click', 'keydown', 'touchstart'].forEach(event => {
  document.addEventListener(event, () => {
    musicManager.unlockAudio();
  }, { once: true });
});

window.addEventListener('beforeunload', () => {
  musicManager.destroy();
});
```

Use the patterns from 03-RESEARCH.md Pattern 1. Accept both `audio/mp3` and `audio/mpeg` MIME types. Do NOT use progress.json for preferences (localStorage only, per research recommendation).
