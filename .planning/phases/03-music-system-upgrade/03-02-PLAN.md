<objective>
Build the music control UI (track selection, volume slider, toggle, custom upload) that lets students interact with the BackgroundMusicManager engine from Plan 01.

Purpose: Without UI controls, the music engine is invisible to students. This plan makes music discoverable, controllable, and personalizable - covering MUS-02 (track choice), MUS-05 (preferences), and MUS-06 (custom MP3s).

**SCOPE CLARIFICATION:** This phase adds background music UI to the **web portal onboarding experience** (web/terminal.html) only. Students spend 5-20 minutes practicing terminal commands in the web portal before entering the actual CLI teaching platform. This music system enhances that onboarding experience. The main CLI teaching sessions (where students complete the 15-module curriculum via `claude` command) currently use afplay for event sounds only. CLI background music is planned for a future phase (Phase 5+).

Output: A collapsible music panel in the terminal page with track cards, volume control, on/off toggle, and MP3 upload capability. All preferences persist in localStorage.
</objective>

Add the following functions and event wiring to the BOTTOM of `web/js/music-system.js` (after the existing BackgroundMusicManager class and initialization code from Plan 01):

**1. Panel toggle (expand/collapse):**
```javascript
function initMusicPanel() {
  const toggle = document.getElementById('music-panel-toggle');
  const body = document.getElementById('music-panel-body');
  const expandIcon = document.getElementById('music-expand-icon');

  toggle.addEventListener('click', () => {
    body.classList.toggle('expanded');
    expandIcon.textContent = body.classList.contains('expanded') ? '\u25B2' : '\u25BC';
  });
}
```

**2. Volume slider wiring:**
Connect `#volume-slider` input event to `musicManager.setVolume(parseFloat(value))`. Update `#volume-value` text to show percentage (Math.round(value * 100) + '%').

**3. Toggle button wiring:**
Connect `#music-toggle-btn` click to `musicManager.toggle()`. Update button text to show ON/OFF. Update `#music-now-playing` to show current track title or "Music: Off".

**4. Track library rendering - `renderMusicLibrary()`:**
- Fetch metadata.json (or use musicManager.trackLibrary if already loaded)
- For each track, create a `.music-track-card` div with title and mood
- Add click handler that calls `musicManager.switchTrack(trackId)`
- Mark active track with `.active` class
- Also render any custom tracks from localStorage `custom_tracks` key
- Call this function after musicManager.init() resolves

**5. Custom MP3 upload handler - `setupCustomUpload()`:**
- Listen on `#custom-music-upload` change event
- Validate file type: accept `audio/mp3`, `audio/mpeg`, `audio/wav`
- Validate file size: max 5MB (5 * 1024 * 1024 bytes)
- Show error in `#music-upload-status` if validation fails
- On valid file:
  - Read as data URL via FileReader
  - Store in localStorage under `custom_tracks` key (JSON object keyed by `custom-{timestamp}`)
  - Each entry: `{ title: filename_without_extension, dataUrl: base64, mood: 'custom', uploadedAt: ISO_date }`
  - Re-render track library to show new track
  - Show success message in `#music-upload-status`
  - Clear the file input value (so same file can be re-uploaded)
- Handle localStorage QuotaExceededError gracefully (show "File too large for browser storage")

**6. Now-playing status updates:**
Add a method `updateNowPlaying(trackId)` that:
- Updates `#music-now-playing` text to show track title
- Updates `.music-track-card.active` class
- Called whenever track switches or toggle changes

**7. Preferences restoration on load:**
After init, restore UI state from preferences:
- Set volume slider value and text
- Set toggle button state (ON/OFF)
- Mark active track card
- Update now-playing text

**8. Wire everything in DOMContentLoaded:**
Update the existing DOMContentLoaded listener to also call:
```javascript
musicManager.init().then(() => {
  initMusicPanel();
  renderMusicLibrary();
  setupCustomUpload();
  restoreUIState();
});
```

Make the init() method return a Promise (or make it async) if not already.

**Important notes:**
- Use event delegation for track card clicks (single listener on `#music-track-list`)
- Do NOT create Howl instances for custom tracks until they are selected (lazy loading)
- For custom tracks, pass the dataUrl directly as Howl src: `src: [dataUrl]`
- The "now playing" display should update on switchTrack completion (after fade-in starts)

Also add a helper method to BackgroundMusicManager that allows future integration of event sounds in the web portal (optional, for future use):

```javascript
// Optional: briefly lower background music for event emphasis
// Future use: if web portal adds celebration events (quest complete, etc.)
duckForEvent(durationMs = 2000) {
  if (!this.currentTrack || !this.preferences.enabled) return;
  const originalVolume = this.preferences.volume;
  const duckedVolume = originalVolume * 0.3; // Drop to 30% of current
  this.currentTrack.fade(originalVolume, duckedVolume, 300);
  setTimeout(() => {
    if (this.currentTrack) {
      this.currentTrack.fade(duckedVolume, originalVolume, 500);
    }
  }, durationMs);
}
```

This method is NOT required for Phase 3, but makes future event sound integration easier if web portal adds celebration effects.
