  <action>
Add a new section "## 15. Module Challenges (Test-Out System)" to CLAUDE.md after Section 14 (Web Onboarding Awareness).

This section should contain:

1. **Overview** explaining the test-out system:
   - Available for Modules 2-7 after Module 1 completion
   - Triggered by `/challenge` command at module start
   - 5-10 minute validation proving existing knowledge
   - Full XP/badge parity with lesson path
   - Unlimited retries with kind feedback

2. **Challenge Trigger Pattern:**
   - Command: `/challenge` (only valid at module start, not mid-module)
   - Prerequisite: Module 1 complete
   - Response: Begin challenge validation for current module

3. **Module 2 Challenge: Installing Claude Code**
   - **Core competencies tested:**
     - Understanding API keys and authentication
     - npm global installation
     - First Claude Code launch
   - **Validation scenarios (3 total, ~7 min):**
     - Scenario 1 (Automated): Verify Claude Code is installed (`claude --version`)
     - Scenario 2 (Conversational): "What does the -g flag mean in npm install -g? Why is it needed for Claude Code?"
     - Scenario 3 (Practical): "Show me how you'd check if a global npm package is installed" (student demonstrates, Claude verifies)
   - **Pass criteria:** All 3 scenarios pass
   - **Badge on pass:** Setup Champion

4. **Module 3 Challenge: First Conversations + Class Selection**
   - **Core competencies tested:**
     - File creation via conversation
     - Specific vs vague prompts
     - Understanding Claude Code's capabilities (what it CAN and CAN'T do)
   - **Validation scenarios (4 total, ~8 min):**
     - Scenario 1 (Practical): "Create a file called challenge-test.txt with specific content" (Claude verifies file exists with correct content)
     - Scenario 2 (Conversational): "What's the difference between asking Claude Code 'make a webpage' vs giving specific requirements?"
     - Scenario 3 (Conversational): "Can Claude Code check the weather? Why or why not?"
     - Scenario 4 (Class Selection): Must select class if not already selected (cannot skip this)
   - **Pass criteria:** Scenarios 1-3 pass, class selected
   - **Badge on pass:** First Contact

5. **Module 4 Challenge: Claude Code Models**
   - **Core competencies tested:**
     - Understanding model differences (Haiku/Sonnet/Opus)
     - Knowing when to use each model
     - Switching models via /model command
   - **Validation scenarios (3 total, ~6 min):**
     - Scenario 1 (Automated): Verify student can switch models (`/model haiku`, `/model sonnet`)
     - Scenario 2 (Conversational): "Describe when you'd use Haiku vs Opus. Give an example task for each."
     - Scenario 3 (Conversational): "Which model is the default and why is it a good default?"
   - **Pass criteria:** All 3 scenarios pass
   - **Badge on pass:** Model Master

Include validation approach notes:
- Conversational validation accepts paraphrasing - test understanding, not memorization
- Automated checks use Bash tool to verify system state
- Practical demonstrations verify actual capability, not just knowledge

Include timing guidance:
- Each challenge should complete in 5-10 minutes
- If taking longer than 10 min, gently suggest lesson path may be better fit
