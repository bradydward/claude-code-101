  - Claude verifies plan mode is off
  - Pass if: Student demonstrates understanding of toggle/exit

**Pass criteria:** All 3 scenarios pass

**Module 7 Challenge: Technical Foundations**

**Core competencies tested:**
- Understanding JSON structure
- Recognizing common file types
- File management (copy, move, delete)
- Understanding paths (absolute vs relative)
- Basic terminal power moves
- Reading error messages

**Validation scenarios (5 total, ~10 min):**

- **Scenario 1 (Practical):** "Create a valid JSON file with an object containing a name string and age number."
  - Claude verifies: file created, JSON is valid, has correct structure
  - Pass if: Valid JSON with string and number properties

- **Scenario 2 (Conversational):** "What's the difference between .md and .json files? When would you use each?"
  - Pass if: Student explains Markdown is for human-readable docs, JSON is for structured data

- **Scenario 3 (Conversational):** "What's the difference between an absolute path and a relative path? Give an example of each."
  - Pass if: Student explains absolute starts from root (/ or ~), relative starts from current location
  - Example: `/Users/name/Desktop` (absolute) vs `./Desktop` or `Desktop` (relative)

- **Scenario 4 (Practical):** "Show me how you'd copy a file to a new location using the terminal."
  - Student demonstrates `cp source destination` pattern
  - Pass if: Student can execute or explain the cp command correctly

- **Scenario 5 (Conversational):** "If you see 'Permission denied' error, what does it usually mean and what would you try?"
  - Pass if: Student mentions file permissions, ownership, or using sudo (with caution note)

**Pass criteria:** At least 4 of 5 scenarios pass (this module is broader, slight leniency)
**Badge on pass:** Tech Foundation
**Topic summary for announcement:** "JSON, file types, paths, file management, and reading error messages"
